Architecture Overview

Understand how Bedrock Agent → Action Group → Lambda → Services connect

💡
A Bedrock Agent orchestrates reasoning. When it needs to do something (query a database, call an API, evaluate an image), it invokes an Action Group. Each Action Group maps to a Lambda function that calls real AWS services and returns structured results.

// IAM

Create IAM Roles

Two roles needed: one for the Agent, one for Lambda

⚠️
You need two separate IAM roles: (A) a role Bedrock assumes to invoke your Lambda, and (B) a role Lambda assumes to call DynamoDB, S3, SNS, etc. Don't mix them.
A

Bedrock Agent Execution Role

Trusted by bedrock.amazonaws.com — allows agent to invoke Lambda & call Bedrock models

B

Lambda Execution Role

Trusted by lambda.amazonaws.com — grants access to DynamoDB, S3, SNS, Bedrock InvokeModel

C

Lambda Resource Policy

Grant bedrock.amazonaws.com permission to invoke the Lambda function (lambda:InvokeFunction)

iam_roles.sh
bash 
# ── A: Create Bedrock Agent Trust Role ──────────────────────────────────

AGENT_ROLE_NAME="AmazonBedrockExecutionRoleForAgents_FlexPLM"

aws iam create-role \
  --role-name "$AGENT_ROLE_NAME" \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": { "Service": "bedrock.amazonaws.com" },
      "Action": "sts:AssumeRole",
      "Condition": {
        "StringEquals": {
          "aws:SourceAccount": "YOUR_ACCOUNT_ID"
        }
      }
    }]
  }'

# Attach inline policy — invoke model + invoke Lambda
aws iam put-role-policy \
  --role-name "$AGENT_ROLE_NAME" \
  --policy-name "BedrockAgentPolicy" \
  --policy-document '{
    "Version": "2012-10-17",
    "Statement": [
      {
        "Effect": "Allow",
        "Action": "bedrock:InvokeModel",
        "Resource": "arn:aws:bedrock:us-east-1::foundation-model/anthropic.claude-sonnet-4*"
      },
      {
        "Effect": "Allow",
        "Action": "lambda:InvokeFunction",
        "Resource": "arn:aws:lambda:us-east-1:YOUR_ACCOUNT_ID:function:flexplm-eval-handler"
      }
    ]
  }'

# ── B: Lambda Execution Role ──────────────────────────────────────────────
aws iam create-role \
  --role-name "FlexPLM-EvalLambdaRole" \
  --assume-role-policy-document '{"Version":"2012-10-17","Statement":[{"Effect":"Allow","Principal":{"Service":"lambda.amazonaws.com"},"Action":"sts:AssumeRole"}]}'

aws iam attach-role-policy \
  --role-name "FlexPLM-EvalLambdaRole" \
  --policy-arn "arn:aws:iam::aws:policy/service-role/AWSLambdaBasicExecutionRole"

aws iam attach-role-policy \
  --role-name "FlexPLM-EvalLambdaRole" \
  --policy-arn "arn:aws:iam::aws:policy/AmazonDynamoDBFullAccess"

# Also attach S3 read + SNS publish inline policy
aws iam put-role-policy \
  --role-name "FlexPLM-EvalLambdaRole" \
  --policy-name "EvalLambdaExtras" \
  --policy-document '{
    "Version": "2012-10-17",
    "Statement": [
      { "Effect":"Allow","Action":["s3:GetObject"],"Resource":"arn:aws:s3:::flexplm-samples/*" },
      { "Effect":"Allow","Action":["sns:Publish"],"Resource":"arn:aws:sns:us-east-1:YOUR_ACCOUNT_ID:flexplm-eval-notifications" },
      { "Effect":"Allow","Action":["bedrock:InvokeModel"],"Resource":"*" }
    ]
  }'

# ── C: Allow Bedrock to invoke the Lambda (resource-based policy) ─────────
aws lambda add-permission \
  --function-name "flexplm-eval-handler" \
  --statement-id "AllowBedrockInvoke" \
  --action "lambda:InvokeFunction" \
  --principal "bedrock.amazonaws.com" \
  --source-account "YOUR_ACCOUNT_ID"

Build the Action Group Lambda

The handler Bedrock calls — routes to functions, calls multiple services, returns structured result

📦
Bedrock sends a fixed event shape. Your Lambda must: (1) parse function + parameters, (2) route to the right internal function, (3) wrap the result in Bedrock's required envelope. All three are non-negotiable.
📦 S3 — sample photos 🤖 Bedrock Vision — image analysis 🗄 DynamoDB — rubric + history 📣 SNS — team notification 🔗 Flex PLM REST API — write memo

lambda_function.py — flexplm-eval-handler
python 
import json, os, boto3, base64, urllib.request
from datetime import datetime, timezone
from decimal import Decimal

# ── AWS clients ────────────────────────────────────────────────────────────
dynamodb        = boto3.resource("dynamodb")
s3              = boto3.client("s3")
sns             = boto3.client("sns")
bedrock_runtime = boto3.client("bedrock-runtime", region_name="us-east-1")

S3_BUCKET       = os.environ["S3_BUCKET"]          # "flexplm-samples"
RUBRIC_TABLE    = os.environ["RUBRIC_TABLE"]        # "flexplm-rubric-criteria"
EVALS_TABLE     = os.environ["EVALS_TABLE"]         # "flexplm-evaluations"
SNS_TOPIC_ARN   = os.environ["SNS_TOPIC_ARN"]
FLEXPLM_API_URL = os.environ["FLEXPLM_API_URL"]    # "https://your-plm.example.com/api"
FLEXPLM_API_KEY = os.environ["FLEXPLM_API_KEY"]


# ══════════════════════════════════════════════════════════════════════════
# ENTRY POINT — Bedrock calls this
# ══════════════════════════════════════════════════════════════════════════
def lambda_handler(event, context):
    print("Bedrock event:", json.dumps(event))

    action_group = event.get("actionGroup", "")
    function     = event.get("function", "")

    # Parse parameter list → dict
    params = {p["name"]: p["value"] for p in event.get("parameters", [])}

    try:
        if function == "evaluate_sample_photos":
            result = evaluate_sample_photos(
                sr_id    = params["sr_id"],
                style_id = params["style_id"],
                s3_keys  = params["s3_keys"].split(","),
            )
        elif function == "get_evaluation_history":
            result = get_evaluation_history(params["style_id"])

        elif function == "issue_decision":
            result = issue_decision(
                sr_id      = params["sr_id"],
                outcome    = params["outcome"],
                memo       = params.get("memo", ""),
                corrections= params.get("corrections", ""),
            )
        else:
            result = {"error": f"Unknown function: {function}"}

    except Exception as e:
        result = {"error": str(e)}

    # ── Required Bedrock response envelope ─────────────────────────────────
    return {
        "messageVersion": "1.0",
        "response": {
            "actionGroup": action_group,
            "function":    function,
            "functionResponse": {
                "responseBody": {
                    "TEXT": {"body": json.dumps(result)}
                }
            }
        }
    }


# ══════════════════════════════════════════════════════════════════════════
# FUNCTION 1 — evaluate_sample_photos
# Flow: S3 → Bedrock Vision → DynamoDB rubric → score → DynamoDB write
# ══════════════════════════════════════════════════════════════════════════
def evaluate_sample_photos(sr_id: str, style_id: str, s3_keys: list) -> dict:

    # Step 1 ─ Download sample images from S3 as base64
    print(f"[1] Fetching {len(s3_keys)} images from S3")
    images_b64 = _fetch_images_from_s3(s3_keys)

    # Step 2 ─ Fetch approved design brief from DynamoDB
    print(f"[2] Loading approved design brief for {style_id}")
    design_brief = _load_design_brief(style_id)

    # Step 3 ─ Load evaluation rubric from DynamoDB
    print("[3] Loading rubric criteria")
    rubric = _load_rubric_criteria()

    # Step 4 ─ Send images + brief + rubric to Claude Vision
    print("[4] Calling Bedrock Claude Vision for analysis")
    analysis = _call_vision_model(images_b64, design_brief, rubric)

    # Step 5 ─ Parse scores and determine overall outcome
    scores  = analysis["scores"]
    outcome = _determine_outcome(scores)

    # Step 6 ─ Persist evaluation record to DynamoDB
    print(f"[6] Saving evaluation → {outcome}")
    eval_id = _save_evaluation(sr_id, style_id, scores, outcome, analysis["summary"])

    # Step 7 ─ Write evaluation memo back to Flex PLM
    print("[7] Posting memo to Flex PLM")
    _post_memo_to_flexplm(sr_id, outcome, analysis["summary"], scores)

    # Step 8 ─ Notify team via SNS
    print("[8] Publishing SNS notification")
    _notify_team(sr_id, style_id, outcome, scores)

    return {
        "eval_id":  eval_id,
        "sr_id":    sr_id,
        "style_id": style_id,
        "outcome":  outcome,       # "PASS" | "CONDITIONAL" | "FAIL"
        "scores":   scores,
        "summary":  analysis["summary"],
        "critical_issues": [s for s in scores.values() if s.get("severity") == "CRITICAL"],
        "next_action": "Advance to Production Handoff" if outcome == "PASS"
                       else "Send correction list to factory" if outcome == "CONDITIONAL"
                       else "Create new Sample Request with root-cause notes",
    }


# ══════════════════════════════════════════════════════════════════════════
# FUNCTION 2 — get_evaluation_history
# Flow: DynamoDB GSI query → return last N evaluations for style
# ══════════════════════════════════════════════════════════════════════════
def get_evaluation_history(style_id: str) -> dict:
    table = dynamodb.Table(EVALS_TABLE)
    resp  = table.query(
        IndexName              = "style_id-index",
        KeyConditionExpression = boto3.dynamodb.conditions.Key("style_id").eq(style_id),
        Limit                  = 10,
        ScanIndexForward       = False,
    )
    return {
        "style_id":   style_id,
        "evaluations": [
            {
                "eval_id":    e["eval_id"],
                "sr_id":      e["sr_id"],
                "outcome":    e["outcome"],
                "created_at": e["created_at"],
                "summary":    e.get("summary", "")[:200],
            }
            for e in resp.get("Items", [])
        ],
    }


# ══════════════════════════════════════════════════════════════════════════
# FUNCTION 3 — issue_decision  (human-confirmed final decision)
# Flow: DynamoDB update → Flex PLM status update → SNS final notice
# ══════════════════════════════════════════════════════════════════════════
def issue_decision(sr_id: str, outcome: str, memo: str, corrections: str) -> dict:
    outcome = outcome.upper()

    # Update DynamoDB eval record
    table = dynamodb.Table(EVALS_TABLE)
    table.update_item(
        Key={"sr_id": sr_id},
        UpdateExpression="SET outcome=:o, memo=:m, confirmed=:c, confirmed_at=:t",
        ExpressionAttributeValues={
            ":o": outcome, ":m": memo,
            ":c": True, ":t": datetime.now(timezone.utc).isoformat()
        }
    )

    # Update Flex PLM SR status via REST API
    status_map = {"PASS": "SAMPLE_APPROVED", "CONDITIONAL": "REVISION_REQUIRED", "FAIL": "SAMPLE_REJECTED"}
    _update_flexplm_sr_status(sr_id, status_map.get(outcome, "UNDER_REVIEW"))

    # Send final SNS
    sns.publish(
        TopicArn = SNS_TOPIC_ARN,
        Subject  = f"[FINAL] SR {sr_id} → {outcome}",
        Message  = json.dumps({"sr_id": sr_id, "outcome": outcome, "memo": memo, "corrections": corrections}),
    )

    return {"sr_id": sr_id, "outcome": outcome, "confirmed": True, "message": f"SR {sr_id} finalized as {outcome}"}
helpers.py — private helper functions called above
python 
def _fetch_images_from_s3(s3_keys: list) -> list:
    """Download images from S3 and return as base64 strings."""
    images = []
    for key in s3_keys:
        key = key.strip()
        obj  = s3.get_object(Bucket=S3_BUCKET, Key=key)
        data = obj["Body"].read()
        images.append({
            "b64":      base64.b64encode(data).decode(),
            "media_type": "image/jpeg" if key.endswith(".jpg") else "image/png",
            "label":    key.split("/")[-1],
        })
    return images


def _load_design_brief(style_id: str) -> dict:
    """Load approved design brief + tech pack from DynamoDB."""
    table = dynamodb.Table(RUBRIC_TABLE)
    resp  = table.get_item(Key={"pk": f"BRIEF#{style_id}"})
    item  = resp.get("Item", {})
    return {
        "silhouette":       item.get("silhouette", "running shoe"),
        "colorways":        item.get("colorways", []),
        "upper_materials":  item.get("upper_materials", []),
        "sole_description": item.get("sole_description", ""),
        "branding_notes":   item.get("branding_notes", ""),
        "closure_type":     item.get("closure_type", "lace"),
    }


def _load_rubric_criteria() -> list:
    """Load standard 10-point evaluation rubric from DynamoDB."""
    table = dynamodb.Table(RUBRIC_TABLE)
    resp  = table.get_item(Key={"pk": "RUBRIC#FOOTWEAR_STANDARD_V2"})
    return resp.get("Item", {}).get("criteria", [
        # fallback default rubric
        {"id": "silhouette",    "label": "Silhouette Accuracy",   "weight": 20},
        {"id": "upper",         "label": "Upper Construction",    "weight": 15},
        {"id": "colorway",      "label": "Colorway Match",        "weight": 15},
        {"id": "sole",          "label": "Sole Profile",          "weight": 15},
        {"id": "stitching",     "label": "Stitching Quality",     "weight": 10},
        {"id": "branding",      "label": "Branding Placement",    "weight": 10},
        {"id": "hardware",      "label": "Hardware & Closures",   "weight": 5},
        {"id": "finishing",     "label": "Edge Finishing",        "weight": 5},
        {"id": "last_fit",      "label": "Last & Fit Profile",   "weight": 3},
        {"id": "labels",        "label": "Labels & Care Tags",   "weight": 2},
    ])


def _call_vision_model(images: list, design_brief: dict, rubric: list) -> dict:
    """
    Call Claude Vision via Bedrock InvokeModel.
    Returns structured scores and a written summary.
    """
    # Build multi-image message content
    content = []
    for img in images:
        content.append({
            "type": "image",
            "source": {"type": "base64", "media_type": img["media_type"], "data": img["b64"]},
        })
        content.append({"type": "text", "text": f"↑ Image: {img['label']}"})

    rubric_text = "\n".join(f"- {c['id']}: {c['label']} (weight {c['weight']}%)" for c in rubric)
    content.append({"type": "text", "text": f"""
You are a senior footwear quality evaluator with 20 years of experience.
Evaluate the sample images against this approved design brief:

DESIGN BRIEF:
{json.dumps(design_brief, indent=2)}

EVALUATION RUBRIC (score each 0-10):
{rubric_text}

For each rubric criterion, provide:
- score: integer 0-10
- severity: OK | MINOR | MAJOR | CRITICAL
- notes: specific observation

Also write a 3-sentence professional evaluation summary.

Respond ONLY in this exact JSON format:
{{
  "scores": {{
    "silhouette":  {{"score": 8, "severity": "OK",    "notes": "Last profile matches approved design"}},
    "colorway":    {{"score": 5, "severity": "MAJOR",  "notes": "Upper color is 2 shades lighter than Pantone 485C spec"}},
    ...
  }},
  "summary": "The sample demonstrates strong construction quality with accurate silhouette reproduction. However, the upper colorway deviates significantly from specification, which is a major concern for brand consistency. Revision is recommended before production approval."
}}"""})

    response = bedrock_runtime.invoke_model(
        modelId     = "anthropic.claude-sonnet-4-20250514",
        contentType = "application/json",
        accept      = "application/json",
        body        = json.dumps({
            "anthropic_version": "bedrock-2023-05-31",
            "max_tokens": 2000,
            "messages": [{"role": "user", "content": content}],
        })
    )
    body = json.loads(response["body"].read())
    text = body["content"][0]["text"]

    # Strip markdown fences if present
    clean = text.replace("```json", "").replace("```", "").strip()
    return json.loads(clean)


def _determine_outcome(scores: dict) -> str:
    """Derive PASS / CONDITIONAL / FAIL from scores dict."""
    has_critical = any(v.get("severity") == "CRITICAL" for v in scores.values())
    major_count  = sum(1 for v in scores.values() if v.get("severity") == "MAJOR")
    avg_score    = sum(v.get("score", 0) for v in scores.values()) / max(len(scores), 1)

    if has_critical or avg_score < 4:  return "FAIL"
    if major_count > 1 or avg_score < 7: return "CONDITIONAL"
    return "PASS"


def _save_evaluation(sr_id, style_id, scores, outcome, summary) -> str:
    import uuid
    eval_id = f"EVAL-{uuid.uuid4().hex[:8].upper()}"
    table   = dynamodb.Table(EVALS_TABLE)
    table.put_item(Item={
        "eval_id":    eval_id, "sr_id": sr_id,
        "style_id":   style_id, "outcome": outcome,
        "scores":     json.dumps(scores),
        "summary":    summary,
        "created_at": datetime.now(timezone.utc).isoformat(),
        "confirmed":  False,
    })
    return eval_id


def _post_memo_to_flexplm(sr_id, outcome, summary, scores):
    """Write evaluation memo back into Flex PLM via REST API."""
    payload = json.dumps({
        "srId":    sr_id, "outcome": outcome,
        "memo":    summary, "scores": scores,
        "source":  "AI_EVALUATION_AGENT",
        "timestamp": datetime.now(timezone.utc).isoformat(),
    }).encode()
    req = urllib.request.Request(
        f"{FLEXPLM_API_URL}/sample-requests/{sr_id}/evaluation",
        data=payload, method="POST",
        headers={"Content-Type": "application/json", "X-Api-Key": FLEXPLM_API_KEY},
    )
    with urllib.request.urlopen(req, timeout=10) as resp:
        return json.loads(resp.read())


def _update_flexplm_sr_status(sr_id, status):
    payload = json.dumps({"status": status}).encode()
    req = urllib.request.Request(
        f"{FLEXPLM_API_URL}/sample-requests/{sr_id}/status",
        data=payload, method="PATCH",
        headers={"Content-Type": "application/json", "X-Api-Key": FLEXPLM_API_KEY},
    )
    with urllib.request.urlopen(req, timeout=10) as resp:
        return json.loads(resp.read())


def _notify_team(sr_id, style_id, outcome, scores):
    icons = {"PASS": "✅", "CONDITIONAL": "⚠️", "FAIL": "❌"}
    sns.publish(
        TopicArn = SNS_TOPIC_ARN,
        Subject  = f"{icons.get(outcome,'?')} Sample Evaluation {outcome} — {sr_id}",
        Message  = json.dumps({
            "sr_id": sr_id, "style_id": style_id,
            "outcome": outcome, "scores": scores,
        }),
    )

Define Action Group Functions

Tell Bedrock what functions exist, their parameters, and when to call them

📋
You define functions in the Bedrock console under Action Groups → Function definitions. The agent uses these descriptions to decide which function to invoke based on the user's request. Write descriptions as if explaining to a smart intern what the function does and when to use it.

agent = bedrock_agent.create_agent(

    agentResourceRoleArn = "arn:aws:iam::YOUR_ACCT:role/AmazonBedrockExecutionRoleForAgents_FlexPLM",

// Integration

Wire Agent to Flex PLM

Invoke the agent from a Flex PLM webhook or from a UI button via API Gateway

🔗
There are two invocation patterns: (A) Webhook-triggered — Flex PLM fires an event when a sample is uploaded → Lambda invokes the agent automatically. (B) Chat UI — evaluator types a natural language request in a sidebar UI → API Gateway → agent.
invoke_agent.py — Pattern A: Webhook trigger
python 
import boto3, json, uuid

bedrock_agent_runtime = boto3.client("bedrock-agent-runtime", region_name="us-east-1")

AGENT_ID    = "AGENTID123"   # from Step 5
AGENT_ALIAS = "ALIAS456"     # from Step 5


def invoke_eval_agent(sr_id: str, style_id: str, s3_keys: list[str]) -> str:
    """
    Called by a Flex PLM webhook when photos are uploaded to an SR.
    Streams the agent's full response and returns it as a string.
    """
    prompt = (
        f"Sample photos have arrived for Sample Request {sr_id} (Style: {style_id}). "
        f"Photos are stored at S3 keys: {', '.join(s3_keys)}. "
        "Please run a full evaluation and provide a recommendation."
    )

    response = bedrock_agent_runtime.invoke_agent(
        agentId      = AGENT_ID,
        agentAliasId = AGENT_ALIAS,
        sessionId    = f"sr-{sr_id}-{uuid.uuid4().hex[:6]}",
        inputText    = prompt,
        enableTrace  = True,   # enables step-by-step reasoning logs in CloudWatch
    )

    # Response is a streaming EventStream — collect all chunks
    full_response = ""
    for event in response["completion"]:
        if "chunk" in event:
            full_response += event["chunk"]["bytes"].decode()
        if "trace" in event:
            # Log agent reasoning steps to CloudWatch
            print("TRACE:", json.dumps(event["trace"], default=str))

    return full_response


# ── Pattern B: Multi-turn chat session (evaluator UI) ─────────────────────
def chat_with_agent(user_message: str, session_id: str) -> str:
    """
    For a chat UI where the evaluator asks follow-up questions.
    Reuse the same session_id to maintain conversation context.
    """
    response = bedrock_agent_runtime.invoke_agent(
        agentId      = AGENT_ID,
        agentAliasId = AGENT_ALIAS,
        sessionId    = session_id,  # ← same session = agent remembers prior turns
        inputText    = user_message,
    )
    full_response = ""
    for event in response["completion"]:
        if "chunk" in event:
            full_response += event["chunk"]["bytes"].decode()
    return full_response


# ── Example webhook handler (Flex PLM calls this) ─────────────────────────
def flexplm_webhook_handler(webhook_event, context):
    """Lambda that Flex PLM calls when sample photos are uploaded."""
    body = json.loads(webhook_event.get("body", "{}"))

    sr_id    = body["srId"]
    style_id = body["styleId"]
    s3_keys  = body["photoS3Keys"]   # list of S3 keys

    result = invoke_eval_agent(sr_id, style_id, s3_keys)
    print(f"Agent response for {sr_id}:\n{result}")

    return {"statusCode": 200, "body": json.dumps({"message": "Evaluation triggered"})}
