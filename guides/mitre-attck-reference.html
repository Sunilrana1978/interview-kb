What the acronym means

Each letter encodes a design principle of the framework itself.

Adversarial
Describes real attacker behavior — not theoretical vulnerabilities. Every entry comes from actual observed intrusions.
Tactics, Techniques
The two primary organizational layers. Tactics = why. Techniques = how. Structured hierarchically with sub-techniques below.
Common Knowledge
A shared, standardized vocabulary so every team — red, blue, intel — can communicate without ambiguity about attacker behaviors.
Why it matters vs. CVE / OWASP

CVE catalogs vulnerabilities in software. OWASP focuses on web application weaknesses. ATT&CK catalogs attacker behaviors — what adversaries actually do once they're in, regardless of which CVE they used to get there. It's behavior-centric, not vuln-centric.

Framework hierarchy

ATT&CK has four levels of specificity. You always read top-down: Tactics contain Techniques, which have Sub-techniques, which are implemented as real-world Procedures by specific threat groups.

Tactics — the "why"
14 high-level categories representing an attacker's immediate goal
e.g. TA0006 — Credential Access
Techniques — the "how"
196 specific methods an attacker uses to achieve a tactic
e.g. T1110 — Brute Force
Sub-techniques — specific variants
411 more granular implementations of a parent technique
e.g. T1110.003 — Password Spraying
Procedures — real-world usage
Specific implementations by named threat actors or malware families
e.g. APT29 used Password Spraying against Office 365 in 2021

How ATT&CK IDs work

Every tactic, technique, and sub-technique has a unique machine-readable ID. Understanding the format lets you instantly recognize what type of entry you're looking at.

T1110.003
T = prefix (T=technique, TA=tactic, S=software, G=group) 1110 = technique number (4 digits) .003 = sub-technique (3 digits, optional)
TA0001 — Tactic
Starts with TA. Represents a high-level attacker goal category.
T1566 — Technique
Starts with T (no A). A specific method to achieve a tactic.
T1566.001 — Sub-technique
T####.### with a dot. A variant of the parent technique.
G0016 — Group
Starts with G. A named threat actor or intrusion set (e.g. APT29 = G0016).

ATT&CK matrices

ATT&CK is organized into domain-specific matrices. Each covers a different attack surface with its own tactic/technique set.

Enterprise
The most commonly used matrix. Covers Windows, macOS, Linux, cloud (AWS/Azure/GCP), network, and containers. 14 tactics, 196 techniques.
Mobile
Android and iOS attack techniques. Covers device access, network-based attacks, and app-layer techniques specific to mobile platforms.
ICS
Industrial Control Systems — SCADA, PLCs, OT networks. Used by critical infrastructure security teams. Covers impair process control, inhibit response.
🔗
The interactive ATT&CK Navigator (attack.mitre.org/matrices) lets you browse the full matrix, layer coverage maps, and export heatmaps for your tooling. Free and browser-based.

Technique deep-dive: Phishing (T1566)

Phishing is one of the most commonly observed Initial Access techniques. Here's how the hierarchy looks fully expanded for a single technique.

T1566 Phishing TA0001 Initial Access

Adversaries send phishing messages to gain access to victim systems. Phishing may include malicious attachments, links to credential harvesting pages, or social engineering to manipulate users into executing attacker-controlled code.

Sub-techniques

T1566.001
Spearphishing Attachment
Targeted email with malicious attachment (Office macros, PDF exploit, ISO file). Most common delivery for ransomware.
T1566.002
Spearphishing Link
Link to attacker-controlled website — credential harvesting page, drive-by download, or OAuth phishing.
T1566.003
Spearphishing via Service
Uses third-party services (Teams, LinkedIn, WhatsApp, Discord) rather than email to deliver the phishing message.
T1566.004
Spearphishing Voice
Vishing — voice/phone call to socially engineer the target into an action or credential disclosure.

A second example: Credential Dumping (T1003)

T1003 OS Credential Dumping TA0006 Credential Access

Adversaries attempt to dump credentials to obtain account login and credential material. Credentials can be used to perform lateral movement and access restricted information.

T1003.001
LSASS Memory
Dump the LSASS process to extract cached credentials. Tools: Mimikatz, ProcDump. Detected by Trellix ENS memory protection rules.
T1003.002
Security Account Manager (SAM)
Extract Windows SAM database for local account hashes. Requires SYSTEM privileges.
T1003.003
NTDS
Dump Active Directory database (ntds.dit). Gives attacker every domain credential. Often the end-goal of lateral movement.
T1003.006
DCSync
Mimics AD replication to pull password hashes without touching disk on DC. Requires domain admin or equivalent rights.

Full attack chain: Ransomware intrusion

A realistic end-to-end ransomware attack mapped to ATT&CK. This is what threat intelligence reports, IR summaries, and SOC playbooks look like in practice. Click any step to understand detection approaches.

1
Reconnaissance
OSINT on target employees
T1591 — Gather victim org info  |  T1589.002 — Email addresses
Attacker scrapes LinkedIn, company website, and data breach databases to identify targets for phishing. No systems touched yet.
2
Initial Access
Spearphishing attachment (malicious Office macro)
T1566.001 — Spearphishing Attachment
Targeted email sent to finance team with an "invoice.xlsm" attachment. User enables macros. Foothold established.
3
Execution
PowerShell downloads payload
T1059.001 — PowerShell  |  T1204.002 — Malicious file
Macro spawns PowerShell to download Cobalt Strike beacon from C2 server. Beacon runs in-memory.
4
Persistence
Scheduled task + registry Run key
T1053.005 — Scheduled Task  |  T1547.001 — Registry Run Keys
Attacker creates a scheduled task and adds Run key to ensure beacon survives reboots.
5
Credential Access
Mimikatz LSASS dump → DCSync
T1003.001 — LSASS Memory  |  T1003.006 — DCSync
Dumps credentials from LSASS. Escalates to domain admin, then runs DCSync to pull all domain password hashes.
6
Lateral Movement
Pass-the-Hash via PsExec
T1550.002 — Pass the Hash  |  T1021.002 — SMB/Windows Admin Shares
Uses domain admin hash to authenticate via SMB and deploy beacon to file servers, backup servers, and Domain Controllers.
7
Impact
Delete VSS + encrypt all drives
T1490 — Inhibit System Recovery  |  T1486 — Data Encrypted for Impact
Deletes Volume Shadow Copies to prevent recovery. Deploys ransomware across all systems simultaneously. Drops ransom note.

How SecOps teams use ATT&CK

ATT&CK provides a common language across the SOC. Here are the four primary operational use cases.

🔍
Threat hunting
Define hypotheses based on specific ATT&CK techniques ("hypothesis: T1003.001 LSASS access on non-DC servers"). Build searches in your SIEM to proactively find attacker behavior before an alert fires.
📊
Coverage mapping / gap analysis
Map your existing tools to ATT&CK technique IDs. The ATT&CK Navigator shows a heatmap of what you can detect and what's blind. Prioritize investment toward high-risk, undetected techniques.
⚔️
Red teaming & purple teaming
Red teams execute specific ATT&CK techniques to test defenses. Purple teams compare what was executed vs. what the blue team detected — using ATT&CK IDs as the shared reference point to close gaps.
🌐
Threat intelligence
Vendor reports and ISACs tag IOCs and actor profiles with ATT&CK IDs. Lets you translate "APT29 is active" into "here are the specific T#### techniques they use — do we detect these?"
📋
Detection rule tagging
Tag every SIEM rule, EDR rule, and playbook with ATT&CK technique IDs. Makes coverage gaps visible and links detections to threat context. Sigma rule format natively supports ATT&CK tags.
📝
Incident reporting
Document incidents with ATT&CK IDs for each adversary action. Creates a structured, reproducible account that feeds back into coverage gaps and detection improvement.

The free ATT&CK Navigator (attack.mitre.org/matrices/enterprise) lets you load detection coverage from tools like Trellix, Splunk, or Microsoft Sentinel as JSON layers. Key SecOps workflow: ENS → ePO → SIEM

ENS detects a technique (e.g. LSASS access) and generates an event tagged with T1003.001. ePO aggregates endpoint events and forwards them to a SIEM. SIEM correlates the ATT&CK-tagged events across your environment to identify attack chains spanning multiple techniques. The ATT&CK ID is the thread connecting detection → alert → investigation → playbook.

Quick reference cheatsheet

Tactic vs Technique
Tactic = goal (why). Technique = method (how). "Credential Access" is a tactic. "Brute Force" is a technique under it.
Sub-technique
A specific variant of a technique. T1110 (Brute Force) has 4 sub-techniques: T1110.001–.004 (Password guessing, Cracking, Spraying, Stuffing).
Procedure
Real-world usage by a specific actor. "APT29 used password spraying against Office 365" is a procedure linked to T1110.003.
ATT&CK Navigator
Free browser tool at attack.mitre.org. Browse the full matrix, build coverage heatmaps, compare threat actor profiles.
Defense Evasion
The largest tactic by technique count (~40). Attackers spend significant effort avoiding detection — disabling AV, obfuscating code, tampering with logs.
Living off the Land (LotL)
Using built-in OS tools (PowerShell, WMI, certutil, mshta) as attack tools. Hard to detect because legitimate tools are used maliciously. Maps to T1218 sub-techniques.
Sigma rules
Open standard for SIEM detection rules. Each Sigma rule includes ATT&CK technique tags. Enables sharing detection logic across different SIEM platforms.
MITRE Engenuity / CTID
MITRE's arm that runs independent evaluations of security products against ATT&CK scenarios. Published evaluations show which techniques vendors detect.
