Terraform Production Security Checklist

Verify all items before promoting infrastructure to production

Infrastructure as Code AWS Security Hardening 12 Controls
Security Controls
  • Encryption Storage Encryption at Rest
    Encryption is enabled on all storage resources — S3, RDS, EBS — all using KMS customer managed keys.
  • IAM Least Privilege IAM Roles
    IAM roles follow least privilege with no wildcard actions (*) or wildcard resources (*) in any policy statement.
  • Network Restricted Security Groups
    Security groups do not allow 0.0.0.0/0 ingress except on public-facing load balancers. All other ingress is scoped to specific CIDR blocks or security group references.
  • Network Private Subnets for Sensitive Resources
    All sensitive resources — databases, application servers, internal services — are deployed in private subnets with no public IP assignment.
  • Audit Logging and Auditing Enabled
    Logging and auditing are enabled across all layers — CloudTrail for API calls, VPC Flow Logs for network traffic, and S3 access logs for object-level activity.
  • State Secure Remote State
    Terraform state file is encrypted using KMS, versioned with S3 versioning enabled, and locked with DynamoDB to prevent concurrent modifications.
  • Protection Deletion Protection on Critical Resources
    Deletion protection is enabled on databases and critical resources. prevent_destroy = true is set in lifecycle blocks to prevent accidental destruction.
  • Validation Input Validation Rules
    All variables have validation blocks enforcing allowed values, CIDR formats, tag requirements, and instance type whitelists before any resources are created.
  • Validation Preconditions and Postconditions
    Resource-level preconditions verify security requirements before creation. Postconditions verify encryption, access controls, and configuration are correctly applied after creation.
  • Scanning Checkov and tfsec Pass Clean
    Static security scans with Checkov and tfsec return no critical or high severity violations. Any suppressions are documented with explicit justification.
  • Secrets Sensitive Outputs Protected
    Sensitive outputs are marked with sensitive = true. Passwords, connection strings, and API keys are never exposed as plain text outputs.
  • Secrets Secrets from Secrets Manager Only
    All secrets — database passwords, API keys, certificates — are fetched from AWS Secrets Manager or HashiCorp Vault at runtime. Nothing sensitive is hardcoded in tfvars or configuration files.
