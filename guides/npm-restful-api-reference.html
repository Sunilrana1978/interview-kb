SERVER

Server / Framework

The foundation of your API. These frameworks handle HTTP routing, middleware, and request/response lifecycle.

$ express
npm install express @types/express

The most widely used Node.js web framework. Minimal and un-opinionated — you wire together exactly what you need using middleware. Massive ecosystem of plugins and community support.

COMMON USE CASES
REST APIs Routing Middleware pipeline Static file serving
TypeScript · Basic Express REST API
import express, { Request, Response } from 'express';

const app = express();

// Parse JSON request bodies
app.use(express.json());

// GET all users
app.get('/users', async (req: Request, res: Response) => {
  const users = await getAllUsers();
  res.json(users);
});

// POST create user
app.post('/users', async (req: Request, res: Response) => {
  const { name, email } = req.body;
  const user = await createUser({ name, email });
  res.status(201).json(user);
});

// GET user by ID
app.get('/users/:id', async (req: Request, res: Response) => {
  const user = await getUserById(req.params.id);
  if (!user) return res.status(404).json({ error: 'Not found' });
  res.json(user);
});

app.listen(3000, () => console.log('API running on port 3000'));

Middleware for handling multipart/form-data, primarily used for file uploads. Supports disk storage, memory storage, and custom storage engines like S3.

TypeScript · File Upload with Validation
import multer from 'multer';
import path from 'path';

const storage = multer.diskStorage({
  destination: (req, file, cb) => cb(null, 'uploads/'),
  filename: (req, file, cb) => {
    const unique = Date.now() + '-' + Math.round(Math.random() * 1e9);
    cb(null, unique + path.extname(file.originalname));
  }
});

const upload = multer({
  storage,
  limits: { fileSize: 5 * 1024 * 1024 },  // 5MB limit
  fileFilter: (req, file, cb) => {
    const allowed = ['image/jpeg', 'image/png', 'image/webp'];
    if (allowed.includes(file.mimetype)) cb(null, true);
    else cb(new Error('Only image files allowed'));
  }
});

// Single file upload
app.post('/upload/avatar', upload.single('avatar'), (req, res) => {
  res.json({ filename: req.file?.filename, path: req.file?.path });
});

// Multiple files
app.post('/upload/gallery', upload.array('images', 10), (req, res) => {
  res.json({ files: req.files });
});

VALIDATION

Schema Validation

Validate incoming request data against a defined schema before processing. Reject bad data early with clear error messages.

$ zod
npm install zod

TypeScript-first schema declaration and runtime validation. Schemas double as TypeScript type definitions — no duplication needed. The most popular validation library for TypeScript projects in 2025.

COMMON USE CASES
Request body validation TypeScript type inference Environment variable validation API contracts
TypeScript · Zod Schema + Validation Middleware
import { z } from 'zod';

// Define schema — also infers TypeScript type
const CreateUserSchema = z.object({
  name:     z.string().min(2).max(100),
  email:    z.string().email(),
  age:      z.number().int().min(18).max(120).optional(),
  role:     z.enum(['admin', 'user', 'moderator']).default('user'),
  tags:     z.array(z.string()).max(5).optional(),
});

// TypeScript type is inferred from the schema — no duplication
type CreateUserInput = z.infer<typeof CreateUserSchema>;

// Reusable validation middleware factory
const validate = (schema: z.ZodSchema) => (req, res, next) => {
  const result = schema.safeParse(req.body);
  if (!result.success) {
    return res.status(400).json({
      error: 'Validation failed',
      issues: result.error.flatten().fieldErrors,
    });
  }
  req.body = result.data;  // Replace with parsed/coerced data
  next();
};

// Use in route
app.post('/users', validate(CreateUserSchema), async (req, res) => {
  const user: CreateUserInput = req.body;
  const created = await createUser(user);
  res.status(201).json(created);
});
$ joi
npm install joi

Battle-tested schema description and data validation. Rich API for complex validations including conditional rules, custom validators, and transformations during validation.

COMMON USE CASES
Complex validations Conditional rules Custom error messages
TypeScript · Joi Validation with Custom Messages
import Joi from 'joi';

const userSchema = Joi.object({
  name:     Joi.string().min(2).max(100).required(),
  email:    Joi.string().email().required(),
  password: Joi.string().min(8)
              .pattern(new RegExp('^(?=.*[A-Z])(?=.*[0-9])'))
              .required()
              .messages({
                'string.pattern.base': 'Password must contain uppercase and number'
              }),
  // Conditional: confirmPassword required only when password present
  confirmPassword: Joi.when('password', {
    is:   Joi.exist(),
    then: Joi.string().valid(Joi.ref('password')).required(),
  }),
});

app.post('/register', (req, res) => {
  const { error, value } = userSchema.validate(req.body, { abortEarly: false });
  if (error) return res.status(400).json({ errors: error.details });
  // value is the validated and coerced object
  res.json({ message: 'Valid!', data: value });
});

Decorator-based validation for TypeScript classes. Used heavily with NestJS where DTOs (Data Transfer Objects) are defined as classes with validation decorators.

TypeScript · DTO with Decorator Validation

  IsString, IsEmail, IsInt, Min, Max,
  IsOptional, IsEnum, ValidateNested, Length
} from 'class-validator';
import { Type } from 'class-transformer';

class AddressDto {
  @IsString() street: string;
  @IsString() city:   string;
  @IsString() zip:    string;
}

enum UserRole { Admin = 'admin', User = 'user' }

export class CreateUserDto {
  @IsString()
  @Length(2, 100)
  name: string;

  @IsEmail()
  email: string;

  @IsInt()
  @Min(18) @Max(120)
  @IsOptional()
  age?: number;

  @IsEnum(UserRole)
  role: UserRole;

  @ValidateNested()          // Validate nested class
  @Type(() => AddressDto)   // Transform plain object to class
  @IsOptional()

$ date-fns
npm install date-fns

Modular date utility library. Parse, format, compare, and manipulate dates without the bloat of moment.js. Tree-shakeable — import only what you use.

COMMON USE CASES
Parse date strings Format for DB storage Date arithmetic Timezone handling
TypeScript · Date Parsing and Transformation
import { parseISO, format, addDays, isAfter, startOfDay } from 'date-fns';

app.post('/bookings', (req, res) => {
  const { checkIn, checkOut } = req.body;

  // Parse ISO string from request to Date object
  const checkInDate  = parseISO(checkIn);
  const checkOutDate = parseISO(checkOut);

  // Validate: check-out must be after check-in
  if (!isAfter(checkOutDate, checkInDate)) {
    return res.status(400).json({ error: 'Check-out must be after check-in' });
  }

  // Add buffer day for cleaning
  const cleaningDay = addDays(checkOutDate, 1);

  // Format for database storage
  const booking = {
    checkIn:     format(checkInDate,  'yyyy-MM-dd'),
    checkOut:    format(checkOutDate, 'yyyy-MM-dd'),
    cleaningDay: format(cleaningDay,   'yyyy-MM-dd'),
    createdAt:   new Date().toISOString(),
  };

  res.json(booking);
});

$ validator
npm install validator @types/validator

String validation and sanitization functions. Validates emails, URLs, credit cards, IP addresses, and more. Also provides sanitizers to escape, trim, and normalize strings.

COMMON USE CASES
Email validation URL validation String escaping Normalize whitespace
TypeScript · Validate and Sanitize Strings
import validator from 'validator';

app.post('/contact', (req, res) => {
  let { email, website, message, phone } = req.body;

  // Normalize and validate email
  email = validator.normalizeEmail(email) || '';
  if (!validator.isEmail(email))
    return res.status(400).json({ error: 'Invalid email' });

  // Validate URL
  if (website && !validator.isURL(website, { protocols: ['https'] }))
    return res.status(400).json({ error: 'URL must use HTTPS' });

  // Sanitize: escape HTML entities in message
  message = validator.escape(validator.trim(message));
  // "Hello <world>" → "Hello &lt;world&gt;"

  // Validate phone number
  if (phone && !validator.isMobilePhone(phone, 'any'))
    return res.status(400).json({ error: 'Invalid phone number' });

  res.json({ email, website, message, phone });
});

$ bcryptjs
npm install bcryptjs @types/bcryptjs

Hash passwords using bcrypt — a slow, salted hashing algorithm designed specifically for passwords. Never store plain-text passwords. bcryptjs is the pure-JS version (no native dependencies).

COMMON USE CASES

import bcrypt from 'bcryptjs';

  const { name, email, password } = req.body;

  // Cost factor 12: ~250ms — slow enough to resist brute force
  const hashedPassword = await bcrypt.hash(password, 12);

// Login: compare input with stored hash
app.post('/auth/login', async (req, res) => {
  const { email, password } = req.body;
  const user = await db.users.findByEmail(email);

  if (!user) return res.status(401).json({ error: 'Invalid credentials' });

  const isMatch = await bcrypt.compare(password, user.password);
  if (!isMatch) return res.status(401).json({ error: 'Invalid credentials' });

  // Issue JWT token...
  res.json({ message: 'Login successful' });
});

Promise-based HTTP client for making outbound API calls from your server — calling third-party APIs, other microservices, or webhooks. Includes interceptors, automatic JSON parsing, and timeout support.

COMMON USE CASES
Third-party API calls Service-to-service Webhooks Request interceptors
TypeScript · Axios Instance with Interceptors
import axios from 'axios';

// Create a configured instance for a specific service
const paymentApi = axios.create({
  baseURL: 'https://api.stripe.com/v1',
  timeout: 10000,
  headers: { Authorization: `Bearer ${process.env.STRIPE_KEY}` }
});

// Request interceptor: add trace ID to every request
paymentApi.interceptors.request.use((config) => {
  config.headers['X-Trace-ID'] = generateTraceId();
  return config;
});

// Response interceptor: normalize errors
paymentApi.interceptors.response.use(
  (response) => response.data,
  (error) => {
    logger.error('Payment API error', { status: error.response?.status });
    throw new PaymentError(error.response?.data?.message);
  }
);

// Usage in service
async function chargeCustomer(amount: number, customerId: string) {
  const charge = await paymentApi.post('/charges', {
    amount, currency: 'usd', customer: customerId
  });
  return charge;
}
$ uuid
npm install uuid @types/uuid

Generate RFC-compliant universally unique identifiers. v4 (random) is most common for entity IDs. v7 (time-ordered) is increasingly preferred for database primary keys as it's sortable.

COMMON USE CASES
Entity IDs Request trace IDs Idempotency keys Session tokens
TypeScript · UUID Generation Patterns
import { v4 as uuidv4, v7 as uuidv7, validate } from 'uuid';

// v4: random UUID — great for user IDs, order IDs
const userId   = uuidv4();  // '9b1deb4d-3b7d-4bad-9bdd-2b0d7b3dcb6d'

// v7: time-ordered UUID — better for DB primary keys (sortable, indexed)
const recordId = uuidv7();  // Starts with current timestamp prefix

// Validate a UUID from request params
app.get('/users/:id', (req, res, next) => {
  if (!validate(req.params.id)) {
    return res.status(400).json({ error: 'Invalid user ID format' });
  }
  next();
});

// Add trace ID to all requests for logging
app.use((req, res, next) => {
  req.traceId = uuidv4();
  res.setHeader('X-Request-ID', req.traceId);
  next();
});
