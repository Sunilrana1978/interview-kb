Store Production Persistence & Distributed State

In testing environments, ADK uses InMemorySessionService to hold conversation variables. For resilient production deployments, you must configure a persistent database adapter (such as PostgreSQL or Spanner) using the DatabaseSessionService.

1. Configuring Persistent DatabaseSessionService

By connecting the ADK runtime to a managed SQL store, conversation state, context memory keys, and execution trajectories survive runtime crashes and can scale horizontally across server instances:

persistence_config.py 
from google.adk.services import DatabaseSessionService
from google.adk.runtime import AgentRuntime
from google.adk.agents import Agent

# Define DB connection parameter string for Cloud SQL / Spanner
db_connection_uri = "postgresql+pg8000://user:pass@/dbname?host=/cloudsql/project:region:instance"

# Initialize persistent state store
session_service = DatabaseSessionService(
    connection_uri=db_connection_uri,
    pool_size=10,
    max_overflow=20
)

# Attach session store directly to the ADK Runtime config
runtime = AgentRuntime(
    session_service=session_service,
    default_model="gemini-2.5-flash"
)

2. State Access Isolation

Always use ToolContext to write and read variables within tool call functions. Never mutate application global variables, as tool calls run concurrently across multiple threads/sessions:

tools/isolated_state.py 
from google.adk.tools import ToolContext

def retrieve_client_history(client_id: str, ctx: ToolContext) -> dict:
    """Retrieves customer transaction records safely.

    Args:
        client_id: Client identifier string.
        ctx: Thread-isolated ToolContext injected at runtime.
    """
    # Safe read from state store
    cached_payload = ctx.get_state(f"client_{client_id}_history", None)
    
    if cached_payload:
        return cached_payload
        
    # Logic to call database backend...
    fresh_payload = {"status": "loaded", "transactions": []}
    
    # Safe write back
    ctx.set_state(f"client_{client_id}_history", fresh_payload)
    return fresh_payload

2. Implementing Workflow Approvals

Define workflow paths that explicitly branch or pause until approval states are satisfied:

hitl_workflow.py 
from google.adk import Agent, Workflow
from google.adk.workflows import RequestInput

# Define execution nodes
transaction_parser = Agent(
    name="transaction_parser",
    instruction="Extract transfer amount and target account info."
)

execute_transfer = Agent(
    name="execute_transfer",
    instruction="Execute bank transaction using parsed parameters."
)

# Human validation placeholder gate
approval_gate = RequestInput(
    name="approval_gate",
    prompt_message="Please review the transaction parameters and confirm: (Y/N)"
)

# Construct graph: Pause at gate before committing critical transfers
payment_workflow = Workflow(
    name="payment_workflow",
    edges=[
        ("START", transaction_parser),
        (transaction_parser, approval_gate),
        (approval_gate, execute_transfer),
        (execute_transfer, "END")
    ]
)

Eval Agent Evaluation & CI/CD Testing

Production agent models require continuous quality validation to prevent regressions. Google ADK provides the evaluation framework to measure semantic correctness and trace tool execution paths.

1. Tool Trajectory Evaluation

Unlike standard language evaluation (like ROUGE or BLEU), Tool Trajectory Evaluation tests that the agent selects the correct tools, calls them in the exact order requested, and passes valid arguments:

eval_script.py 
from google.adk.evaluation import AgentEvaluator, GoldenTestCase
from agents.weather import weather_agent

# Create test cases asserting semantic outputs AND exact tool trajectories
test_cases = [
    GoldenTestCase(
        input_query="What's the temp in Boston?",
        expected_output="68F",
        expected_tool_trajectory=[
            # Asserts that 'get_temperature' is called with 'Boston'
            ("get_temperature", {"city": "Boston"})
        ]
    )
]

evaluator = AgentEvaluator(agent=weather_agent)
report = evaluator.run(test_cases)
print(f"Overall Accuracy: {report.accuracy_score * 100}%")

2. Containerization (Dockerfile)

Deploy your ADK application using a standardized production Python environment:

Dockerfile 
FROM python:3.10-slim

WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

# Run the ADK Runtime Server listening on target ports
EXPOSE 8080
CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "8080"]

3. VPC Service Controls & IAM

To secure enterprise database operations and Gemini connections:

  • Least Privilege IAM Roles: Run Cloud Run containers under a custom Service Account restricted to the roles/aiplatform.user role (to request Vertex AI models) and roles/spanner.databaseUser (for persistence).
  • VPC Network egress: Direct database requests inside a private VPC network using Serverless VPC Access Connectors, blocking external internet vectors to your database.
