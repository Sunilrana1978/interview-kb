// google-genai SDK — route comparison

One SDK.
Two routes to Gemini.

The Developer API and Vertex AI expose the same models through the same google-genai client — what changes is auth, governance, and what's wired up for RAG retrieval and agent orchestration. Toggle the schematic below to trace a request through each route.

request path
GEMINI_API_KEY → generativelanguage.googleapis.com

01 — the split

What actually changes underneath

Same model family, same SDK calls. These are the parts of the contract that differ once you pick a lane.

  DEVELOPER API VERTEX AI
Auth Single GEMINI_API_KEY env var, tied to a Cloud project behind the scenes Application Default Credentials or service account, managed through Cloud IAM
Client init genai.Client(api_key=...) genai.Client(vertexai=True, project=, location=)
Governance None built in — no audit logs, no VPC controls IAM roles, VPC Service Controls, CMEK, Cloud Logging / Monitoring
Data use Free-tier inputs/outputs may be used to improve Google's models Not used for model training
Quota model Per-key rate tiers (RPM / TPM / RPD), pay per call Dynamic shared quota, or pre-purchased provisioned throughput
Grounding tools Google Search grounding, function calling, code execution All of Developer API's tools, plus Vertex AI Search grounding over your own indexed data
Agent tooling Function calling + Interactions API for multi-turn state Adds Agent Designer, ADK, and a managed Agent Engine runtime with Sessions / Memory Bank
Best fit Solo / small-team builds, prototypes, side projects not already on GCP Org already on Google Cloud, needs audit trail, data residency, or SLA

04 — the call

Which one for AskLore

Developer API — start here

  • Not on GCP today — your stack is AWS-native (OpenSearch, Titan, Bedrock)
  • Small team target market doesn't need IAM/audit overhead
  • API key setup takes minutes, not a Cloud project
  • Good enough free tier for prototyping the Bedrock → Gemini swap

Vertex AI — revisit later if

  • A customer requires data-residency or "not used for training" guarantees
  • You need an audit trail for compliance-minded engineering teams
  • You outgrow per-key rate limits and want provisioned throughput
  • You want managed session memory instead of building it yourself
Both routes run on the same google-genai Python/JS SDK — migrating between them later is a client-init change, not a rewrite. Verify current pricing, quotas, and model availability against ai.google.dev before committing, as these shift often.
