📄 alb-composite-alarm.yaml
# ══════════════════════════════════════════════════════
# Example 1: ALB-backed app — Composite Alarm (Metric Math)
# Signals: 5xx errors + p99 latency + Route53 health check
# Notification: SNS → Email (on-call engineer)
# ══════════════════════════════════════════════════════

AWSTemplateFormatVersion: "2010-09-09"
Description: ALB Composite Alarm using Metric Math — single alarm, no child alarms

Parameters:
  AppName:
    Type: String
    Default: my-app
  LoadBalancerName:
    Type: String
    Description: Full ALB resource name (e.g. app/my-alb/abc1234567)
  HealthCheckId:
    Type: String
    Description: Route53 Health Check ID
  OnCallEmail:
    Type: String
    Description: On-call engineer email address

Resources:

    # SNS Topic — single notification target
  AlertTopic:
    Type: AWS::SNS::Topic
    Properties:
      TopicName: !Sub "${AppName}-outage-alert"
      Subscription:
        - Protocol: email
          Endpoint: !Ref OnCallEmail

    # Single composite alarm — Metric Math expression
  OutageAlarm:
    Type: AWS::CloudWatch::Alarm
    Properties:
      AlarmName: !Sub "${AppName}-ALB-OUTAGE"
      AlarmDescription: High 5xx + high latency + health check down simultaneously

      Metrics:
                # m1 — 5xx error count from ALB
        - Id: m1
          MetricStat:
            Metric:
              Namespace: AWS/ApplicationELB
              MetricName: HTTPCode_Target_5XX_Count
              Dimensions:
                - Name: LoadBalancer
                  Value: !Ref LoadBalancerName
            Period: 60
            Stat: Sum
          ReturnData: false

                # m2 — p99 latency from ALB
        - Id: m2
          MetricStat:
            Metric:
              Namespace: AWS/ApplicationELB
              MetricName: TargetResponseTime
              Dimensions:
                - Name: LoadBalancer
                  Value: !Ref LoadBalancerName
            Period: 60
            Stat: p99
          ReturnData: false

                # m3 — Route53 health check (1=healthy, 0=down)
        - Id: m3
          MetricStat:
            Metric:
              Namespace: AWS/Route53
              MetricName: HealthCheckStatus
              Dimensions:
                - Name: HealthCheckId
                  Value: !Ref HealthCheckId
            Period: 60
            Stat: Minimum
          ReturnData: false

                # expr — fires when ALL three breach at once
        - Id: expr
          Expression: "IF(m1 > 5 AND m2 > 0.5 AND m3 < 1, 1, 0)"
          Label: ALBOutageSignal
          ReturnData: true

      ComparisonOperator: GreaterThanOrEqualToThreshold
      Threshold: 1
      EvaluationPeriods: 2
      TreatMissingData: notBreaching
      AlarmActions:
        - !Ref AlertTopic
      OKActions:
        - !Ref AlertTopic

Outputs:
  AlarmName:
    Value: !Ref OutageAlarm
  SNSTopicArn:
    Value: !Ref AlertTopic

🚀 Deploy Commands

STEP 1
Save the template
Save the YAML above as alb-composite-alarm.yaml in your working directory.
STEP 2
Confirm AWS credentials
Run aws sts get-caller-identity to verify you're authenticated to the correct account and region.
STEP 3
Deploy the stack
Run the deploy command below. CloudFormation creates the SNS topic and alarm in one operation.
STEP 4
Confirm email subscription
Check the on-call inbox for an AWS SNS confirmation email and click the link to activate alerts.
Terminal — AWS CLI
# ── Verify AWS identity first ────────────────────────── aws sts get-caller-identity # ── Deploy ALB composite alarm stack ─────────────────── aws cloudformation deploy \ --template-file alb-composite-alarm.yaml \ --stack-name my-app-alb-alarms \ --region us-east-1 \ --parameter-overrides \ AppName=my-app \ LoadBalancerName=app/my-app-alb/abc1234567890 \ HealthCheckId=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx \ OnCallEmail=oncall@yourcompany.com # ── Verify stack deployed successfully ───────────────── aws cloudformation describe-stacks \ --stack-name my-app-alb-alarms \ --query "Stacks[0].StackStatus" # ── Check the alarm state ─────────────────────────────── aws cloudwatch describe-alarms \ --alarm-names my-app-ALB-OUTAGE \ --query "MetricAlarms[0].StateValue" # ── Delete stack if no longer needed ─────────────────── aws cloudformation delete-stack \ --stack-name my-app-alb-alarms
⚠ Note: Replace app/my-app-alb/abc1234567890 with your actual ALB resource name — find it in the AWS Console under EC2 → Load Balancers → select your ALB → copy the ARN suffix after loadbalancer/. The Route53 HealthCheckId is found under Route53 → Health Checks.

🟣 PagerDuty Integration via SNS

Instead of an email subscription, add PagerDuty's HTTPS endpoint as an SNS subscriber. PagerDuty provides a unique Events API v2 integration URL per service.

In the SNS subscription, set Protocol: https and Endpoint: https://events.pagerduty.com/integration/<YOUR_KEY>/enqueue. PagerDuty auto-confirms the SNS subscription and routes alerts based on your escalation policy.

You can add both PagerDuty and Slack webhooks as separate subscribers on the same SNS topic — one alarm, multiple notification channels.

📄 cloudfront-spa-composite-alarm.yaml
# ══════════════════════════════════════════════════════
# Example 2: SPA on CloudFront + API Gateway + PagerDuty
# Signals: CF 5xx rate + API Gateway errors + Route53 health
#          + CloudFront request drop (silent outage detection)
# Notification: SNS → PagerDuty HTTPS endpoint
# ══════════════════════════════════════════════════════

AWSTemplateFormatVersion: "2010-09-09"
Description: CloudFront SPA Composite Alarm — Metric Math + PagerDuty

Parameters:
  AppName:
    Type: String
    Default: my-spa
  CloudFrontDistributionId:
    Type: String
    Description: CloudFront Distribution ID (e.g. E1ABCDEF123456)
  ApiGatewayName:
    Type: String
    Description: API Gateway name (matches ApiName dimension)
  ApiGatewayStage:
    Type: String
    Default: prod
  HealthCheckId:
    Type: String
    Description: Route53 Health Check ID for your domain
  PagerDutyEndpoint:
    Type: String
    Description: PagerDuty Events API v2 HTTPS URL for your service
        # Format: https://events.pagerduty.com/integration/<KEY>/enqueue

Resources:

    # SNS Topic — routes to PagerDuty (and optionally Slack)
  AlertTopic:
    Type: AWS::SNS::Topic
    Properties:
      TopicName: !Sub "${AppName}-outage-alert"
      Subscription:
                # PagerDuty receives SNS via HTTPS and creates incidents
        - Protocol: https
          Endpoint: !Ref PagerDutyEndpoint
                # Optional: add Slack webhook as second subscriber
        # - Protocol: https
        #   Endpoint: https://hooks.slack.com/services/YOUR/SLACK/WEBHOOK

    # Single composite alarm using Metric Math across 4 signals
  SPAOutageAlarm:
    Type: AWS::CloudWatch::Alarm
    Properties:
      AlarmName: !Sub "${AppName}-SPA-OUTAGE"
      AlarmDescription: >
        CRITICAL: SPA outage detected. CloudFront errors, API Gateway
        failures, and health check down. Page SRE immediately.

      Metrics:
                # m1 — CloudFront 5xx error rate (%)
        # NOTE: CloudFront metrics must use us-east-1 region
        - Id: m1
          MetricStat:
            Metric:
              Namespace: AWS/CloudFront
              MetricName: 5xxErrorRate
              Dimensions:
                - Name: DistributionId
                  Value: !Ref CloudFrontDistributionId
                - Name: Region
                  Value: Global
            Period: 60
            Stat: Average
          ReturnData: false

                # m2 — API Gateway 5xx error count
        - Id: m2
          MetricStat:
            Metric:
              Namespace: AWS/ApiGateway
              MetricName: 5XXError
              Dimensions:
                - Name: ApiName
                  Value: !Ref ApiGatewayName
                - Name: Stage
                  Value: !Ref ApiGatewayStage
            Period: 60
            Stat: Sum
          ReturnData: false

                # m3 — Route53 health check (1=healthy, 0=down)
        - Id: m3
          MetricStat:
            Metric:
              Namespace: AWS/Route53
              MetricName: HealthCheckStatus
              Dimensions:
                - Name: HealthCheckId
                  Value: !Ref HealthCheckId
            Period: 60
            Stat: Minimum
          ReturnData: false

                # m4 — CloudFront total request count
        # Catches silent outages: traffic drops to near-zero
        - Id: m4
          MetricStat:
            Metric:
              Namespace: AWS/CloudFront
              MetricName: Requests
              Dimensions:
                - Name: DistributionId
                  Value: !Ref CloudFrontDistributionId
                - Name: Region
                  Value: Global
            Period: 60
            Stat: Sum
          ReturnData: false

                # expr — Two trigger scenarios:
        # Path A: CF errors + API errors + health check down
        # Path B: Traffic vanished + health check down (silent outage)
        - Id: expr
          Expression: "IF(m1 > 5 AND m2 > 10 AND m3 < 1, 1, IF(m4 < 10 AND m3 < 1, 1, 0))"
          Label: SPAOutageSignal
          ReturnData: true

      ComparisonOperator: GreaterThanOrEqualToThreshold
      Threshold: 1
      EvaluationPeriods: 2
      TreatMissingData: breaching   # Missing CloudFront data = assume down
      AlarmActions:
        - !Ref AlertTopic
      OKActions:
        - !Ref AlertTopic

Outputs:
  AlarmName:
    Value: !Ref SPAOutageAlarm
    Description: Composite alarm for SPA outage detection
  SNSTopicArn:
    Value: !Ref AlertTopic
    Description: SNS topic ARN — add more subscribers (Slack, email) here

🚀 Deploy Commands

STEP 1
Save the template
Save the YAML as cloudfront-spa-composite-alarm.yaml in your working directory.
STEP 2 — CRITICAL
Use us-east-1 region
CloudFront metrics only exist in us-east-1. This stack MUST be deployed there regardless of where your app runs.
STEP 3
Get PagerDuty URL
In PagerDuty → Services → your service → Integrations → Add Integration → Amazon CloudWatch. Copy the HTTPS endpoint URL.
STEP 4
Deploy & confirm
Run the deploy command. PagerDuty auto-confirms the SNS subscription — no manual email confirmation needed.
Terminal — AWS CLI
# ── Verify AWS identity ───────────────────────────────── aws sts get-caller-identity # ── IMPORTANT: CloudFront metrics only exist in us-east-1 # Deploy this stack to us-east-1 regardless of app region # ── Deploy CloudFront SPA composite alarm stack ───────── aws cloudformation deploy \ --template-file cloudfront-spa-composite-alarm.yaml \ --stack-name my-spa-cf-alarms \ --region us-east-1 \ --parameter-overrides \ AppName=my-spa \ CloudFrontDistributionId=E1ABCDEF1234567 \ ApiGatewayName=my-api \ ApiGatewayStage=prod \ HealthCheckId=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx \ PagerDutyEndpoint=https://events.pagerduty.com/integration/YOUR_KEY/enqueue # ── Verify stack status ───────────────────────────────── aws cloudformation describe-stacks \ --stack-name my-spa-cf-alarms \ --region us-east-1 \ --query "Stacks[0].StackStatus" # ── Check alarm state ─────────────────────────────────── aws cloudwatch describe-alarms \ --alarm-names my-spa-SPA-OUTAGE \ --region us-east-1 \ --query "MetricAlarms[0].StateValue" # ── Test alarm manually (force ALARM state) ───────────── aws cloudwatch set-alarm-state \ --alarm-name my-spa-SPA-OUTAGE \ --state-value ALARM \ --state-reason "Manual test — verifying PagerDuty integration" \ --region us-east-1 # ── Reset alarm back to OK after testing ──────────────── aws cloudwatch set-alarm-state \ --alarm-name my-spa-SPA-OUTAGE \ --state-value OK \ --state-reason "Resetting after manual test" \ --region us-east-1 # ── Delete stack if no longer needed ──────────────────── aws cloudformation delete-stack \ --stack-name my-spa-cf-alarms \ --region us-east-1
⚠ CloudFront Distribution ID: Find it in AWS Console → CloudFront → Distributions → the ID column (e.g. E1ABCDEF1234567). Your API Gateway name is under API Gateway → APIs → the API name shown in the list. The set-alarm-state commands let you test the PagerDuty integration without waiting for a real outage.

💡 Key Interview Takeaway

Both examples use a single AWS::CloudWatch::Alarm with a Metric Math Expression entry — no child alarms, no CompositeAlarm resource needed.

The expression is evaluated every period. It returns 1 only when the right combination of signals breach — eliminating noise from individual metric spikes. One alarm fires one SNS message to one on-call channel.

For CloudFront, always remember: CloudFront metrics only exist in us-east-1. Your CloudFormation stack must be deployed there, or you must use a cross-region metric reference.

Quick Deploy Reference
ALB + EC2 — Deploy Command
aws cloudformation deploy \ --template-file alb-composite-alarm.yaml \ --stack-name my-app-alb-alarms \ --region us-east-1 \ --parameter-overrides \ AppName=my-app \ LoadBalancerName=app/my-app-alb/abc1234567890 \ HealthCheckId=YOUR_HEALTHCHECK_ID \ OnCallEmail=oncall@yourcompany.com
CloudFront + SPA — Deploy Command
# Must deploy to us-east-1 — CloudFront metrics only exist there aws cloudformation deploy \ --template-file cloudfront-spa-composite-alarm.yaml \ --stack-name my-spa-cf-alarms \ --region us-east-1 \ --parameter-overrides \ AppName=my-spa \ CloudFrontDistributionId=E1ABCDEF1234567 \ ApiGatewayName=my-api \ ApiGatewayStage=prod \ HealthCheckId=YOUR_HEALTHCHECK_ID \ PagerDutyEndpoint=https://events.pagerduty.com/integration/YOUR_KEY/enqueue
Useful Verification Commands (both stacks)
# Check stack status aws cloudformation describe-stacks \ --stack-name STACK_NAME \ --query "Stacks[0].StackStatus" # Check alarm state aws cloudwatch describe-alarms \ --alarm-names ALARM_NAME \ --query "MetricAlarms[0].StateValue" # Manually trigger alarm to test notifications aws cloudwatch set-alarm-state \ --alarm-name ALARM_NAME \ --state-value ALARM \ --state-reason "Manual test" # Reset alarm after test aws cloudwatch set-alarm-state \ --alarm-name ALARM_NAME \ --state-value OK \ --state-reason "Reset after manual test" # List all CloudWatch alarms in account aws cloudwatch describe-alarms \ --query "MetricAlarms[*].{Name:AlarmName,State:StateValue}" \ --output table # Delete a stack cleanly aws cloudformation delete-stack \ --stack-name STACK_NAME
