Build & Publish a TypeScript Library

Step-by-step guide to create a reusable npm package with a client class, deploy to Artifactory, and let users import and use it.

STEP 01

Initialize the Project

Create a new folder for your library and initialize it with npm. We'll use TypeScript so the library ships with type definitions.

$ terminal
mkdir my-api-library
cd my-api-library
npm init -y
npm install --save-dev typescript tsup rimraf
Note: tsup is a fast, zero-config TypeScript bundler. It produces both CommonJS and ESM outputs automatically.

Your folder structure will look like this:

my-api-library/
  ├── src/
  │   ├── client.ts   // main client class
  │   └── index.ts   // public exports
  ├── tsconfig.json
  ├── package.json
  └── tsup.config.ts
STEP 02

Configure TypeScript

Create a tsconfig.json at the root of your project.

tsconfig.json
{
  "compilerOptions": {
    "target": "ES2019",
    "module": "ESNext",
    "moduleResolution": "bundler",
    "declaration": true,          // generate .d.ts files
    "declarationMap": true,
    "sourceMap": true,
    "strict": true,
    "outDir": "dist",
    "rootDir": "src",
    "esModuleInterop": true,
    "skipLibCheck": true
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist"]
}

Then create the bundler config:

tsup.config.ts
import { defineConfig } from 'tsup'

export default defineConfig({
  entry: ['src/index.ts'],
  format: ['cjs', 'esm'],  // CommonJS + ES Module
  dts: true,              // generate type declarations
  splitting: false,
  sourcemap: true,
  clean: true,             // clean dist/ before each build
  minify: false
})
STEP 03

Write the Client Class

Create src/client.ts. This is your main class that users will instantiate and call methods on.

src/client.ts
// Configuration options the user passes when creating the client
export interface MyClientOptions {
  baseUrl: string
  apiKey: string
  timeout?: number   // optional, defaults to 5000ms
}

// Shape of a User returned by getUser()
export interface User {
  id: string
  name: string
  email: string
}

export class MyClient {
  private baseUrl: string
  private apiKey: string
  private timeout: number

  constructor(options: MyClientOptions) {
    this.baseUrl = options.baseUrl
    this.apiKey  = options.apiKey
    this.timeout = options.timeout ?? 5000
  }

  /** Build shared request headers */
  private getHeaders(): HeadersInit {
    return {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${this.apiKey}`
    }
  }

  /** Fetch a user by ID */
  async getUser(userId: string): Promise<User> {
    const response = await fetch(
      `${this.baseUrl}/users/${userId}`,
      { headers: this.getHeaders() }
    )
    if (!response.ok) {
      throw new Error(`Failed to fetch user: ${response.status}`)
    }
    return response.json() as Promise<User>
  }

  /** Create a new user */
  async createUser(data: Omit<User, 'id'>): Promise<User> {
    const response = await fetch(`${this.baseUrl}/users`, {
      method: 'POST',
      headers: this.getHeaders(),
      body: JSON.stringify(data)
    })
    if (!response.ok) {
      throw new Error(`Failed to create user: ${response.status}`)
    }
    return response.json() as Promise<User>
  }

  /** List all users */
  async listUsers(): Promise<User[]> {
    const response = await fetch(`${this.baseUrl}/users`, {
      headers: this.getHeaders()
    })
    if (!response.ok) {
      throw new Error(`Failed to list users: ${response.status}`)
    }
    return response.json() as Promise<User[]>
  }
}
STEP 04

Create the Public Entry Point

Create src/index.ts. This is the only file users import from — it controls what's public.

src/index.ts
// Re-export everything users need
export { MyClient } from './client'
export type { MyClientOptions, User } from './client'

// Export the version so users can check it at runtime
export const VERSION = '1.0.0'
Tip: Only export what users need. Keep internal helpers private to your src/ folder and don't re-export them here.
STEP 05

Build the Package

Add build scripts to your package.json and run the build.

$ terminal
# Add this to package.json scripts (shown in step 6)
npm run build

# This generates dist/ with:
#   dist/index.js       → CommonJS (for Node.js require)
#   dist/index.mjs      → ES Module (for modern bundlers)
#   dist/index.d.ts     → TypeScript types

After building, your dist/ folder looks like:

dist/
  ├── index.js    // CommonJS bundle
  ├── index.mjs   // ESM bundle
  ├── index.d.ts  // TypeScript declarations
  └── index.js.map
STEP 06

Configure package.json

Update your package.json so npm and bundlers know where to find your code, types, and both module formats.

package.json
{
  "name": "@your-org/my-api-library",   // scoped = org/team prefix
  "version": "1.0.0",
  "description": "Client library for My API",
  "main": "dist/index.js",            // CommonJS entry
  "module": "dist/index.mjs",         // ESM entry
  "types": "dist/index.d.ts",         // TypeScript types
  "exports": {
    ".": {
      "import": "./dist/index.mjs",
      "require": "./dist/index.js",
      "types": "./dist/index.d.ts"
    }
  },
  "files": ["dist"],               // only ship dist/ to npm
  "scripts": {
    "build": "rimraf dist && tsup",
    "prepublishOnly": "npm run build"  // build before publish
  },
  "publishConfig": {
    "registry": "https://your-org.jfrog.io/artifactory/api/npm/npm-local/"
  }
}
Note: The publishConfig.registry points to your Artifactory instance. Replace the URL with your actual Artifactory npm registry URL.
STEP 07

Publish to Artifactory

Authenticate with your Artifactory registry and publish the package.

$ terminal — authenticate
# Option A: Login interactively
npm login --registry=https://your-org.jfrog.io/artifactory/api/npm/npm-local/

# Option B: Set auth token in .npmrc (CI/CD)
echo "//your-org.jfrog.io/artifactory/api/npm/npm-local/:_authToken=YOUR_TOKEN" >> ~/.npmrc
$ terminal — publish
# Build and publish in one step
npm publish --registry=https://your-org.jfrog.io/artifactory/api/npm/npm-local/

# Or just run npm publish — package.json publishConfig handles the registry
npm publish
Versioning: Before publishing a new version, bump the version number using npm version patch (bug fix), npm version minor (new feature), or npm version major (breaking change).
STEP 08

Install & Use the Package

Now any team or app can install your library and start using the client.

$ terminal — install
# Configure Artifactory registry for the scope (one-time)
npm config set @your-org:registry https://your-org.jfrog.io/artifactory/api/npm/npm-local/

# Install the package
npm install @your-org/my-api-library

Then in any TypeScript or JavaScript file:

app.ts — usage example
import { MyClient } from '@your-org/my-api-library'

// 1. Create the client
const client = new MyClient({
  baseUrl: 'https://api.myservice.com',
  apiKey: 'your-api-key-here',
  timeout: 8000
})

// 2. Use its methods
async function main() {
  // Get a single user
  const user = await client.getUser('user-123')
  console.log('User:', user.name)

  // Create a new user
  const newUser = await client.createUser({
    name: 'Jane Doe',
    email: 'jane@example.com'
  })
  console.log('Created:', newUser.id)

  // List all users
  const users = await client.listUsers()
  console.log('Total users:', users.length)
}

main().catch(console.error)
TypeScript autocomplete: Because your library ships .d.ts files, users get full IntelliSense, autocomplete, and type checking in their editor automatically — no extra setup needed.
