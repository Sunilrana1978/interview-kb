AWS Cloud Architect

Key Topics · Architecture Patterns · Best Practices
🏗️ IaC & Terraform 01
  • Modular structure — one module per service
  • Root stack combines all modules
  • Outputs from one → inputs to another
  • Version control + code review for infra
  • Environment parity: dev / staging / prod
  • CI/CD pipeline integration
  • State management tracks real resources
Terraform CloudFormation Modules
⚡ Event-Driven Architecture 02
Flow
  • SMS Source → SQS (buffer & decouple)
  • Lambda #1 → processes & writes to DynamoDB
  • DynamoDB Streams → captures changes in real-time
  • Lambda #2 → consumes stream, publishes to EventBridge
  • EventBridge Rules → route by consumer type
  • Consumers → subscribe to relevant rules
  • Loose coupling — components only know events
  • Lambda auto-scales per message volume
SQS Lambda DynamoDB Streams EventBridge
📊 Monitoring 03
  • CloudWatch Logs per Lambda
  • Custom metrics via Lambda Powertools
  • Alarms on error rate, DLQ depth
  • X-Ray for distributed tracing
  • Structured JSON logs for Insights
  • SQS dead letter queue monitoring
Key Metrics
  • Processing time, error count
  • DynamoDB write latency
  • Lambda cold starts, duration
🛡️ Data Protection 04
  • DynamoDB point-in-time recovery (35 days)
  • Terraform deletion protection flag
  • S3 versioning + MFA delete
  • AWS Backup — centralized mgmt
  • IAM policies block accidental deletes
  • Lifecycle policies for storage tiers
PITR S3 Versioning AWS Backup
💰 Cost Optimization 05
  • Tag all resources in Terraform (env, team, project)
  • Cost Explorer — slice by tag/service
  • Right-size EC2 per environment
  • Lambda memory tuning (cost vs speed)
  • DynamoDB on-demand for dev/unpredictable
  • Reserved instances for prod
  • Shut down non-prod off-hours
  • S3 lifecycle → Glacier for old logs
🔐 Security 06
  • IAM least privilege per service
  • Encryption at rest & in transit
  • VPC isolation + security groups
  • Secrets Manager / Parameter Store
  • No hardcoded credentials in IaC
  • Audit trails via CloudTrail
IAM KMS VPC CloudTrail
🔍 Debug Methodology 07
  • Step 1 — Check CloudWatch Logs for error
  • Step 2 — X-Ray trace full request path
  • Step 3 — Check SQS dead letter queue
  • Step 4 — Verify DynamoDB Streams active
  • Step 5 — Check EventBridge rule matches
  • Structured logs = searchable errors
🧠 Architect Design Thinking 08
  • Scalability — auto-scaling, serverless, stateless design
  • Resilience — multi-AZ, retries, DLQ, circuit breakers
  • Trade-offs — serverless vs containers vs EC2
  • Consistency — eventual vs strong consistency decisions
  • Observability — logs, metrics, traces (3 pillars)
  • Cost awareness — design for efficiency from day one
  • Always anchor answers to real project examples
  • Show the full picture — not just the happy path
